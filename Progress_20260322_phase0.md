## 关键观察

### 1. Bandwidth-bound 再次验证
- 0.8B-8bit (0.80GB) vs 0.8B-bf16 (1.47GB): TPS 比 = 302.7/219.2 = 1.38, 内存比 = 1.47/0.80 = 1.84
- 相同模型不同量化 → TPS 差异主要由权重大小（带宽需求）决定
- 2B-6bit (1.49GB) vs 2B-bf16 (3.58GB): TPS 比 = 214.3/129.4 = 1.66, 内存比 = 3.58/1.49 = 2.40

### 2. 模型规模收益 >> 量化精度收益（H2OAttnScore 结论再确认）
- 4B-4bit: PPL=4.959, Mem=2.34GB → **PPL 效率 = (6.185-4.959)/2.34 = 0.52/GB**
- 2B-bf16: PPL=5.366, Mem=3.58GB → **PPL 效率 = (6.185-5.366)/3.58 = 0.23/GB**
- 大模型+激进量化 >> 小模型+高精度

### 3. 8GB 设备临界点
- 9B-6bit (6.90GB): 在 8GB 设备上可能勉强可行，但留给 KV cache 和系统的空间很小
- 9B-8bit (8.97GB): 超出 8GB → **必须 streaming**
- 27B-4bit (14.34GB): 超出 8GB 约 80% → **极限挑战**

### 4. 27B 的 TPS 上限
- Mac Studio 全驻留: 35.0 TPS
- 这是 27B-4bit 的理论最大值（无任何 streaming 开销）
- 8GB 设备上即使 streaming 能跑，目标是达到这个值的 10-30%（3-10 TPS）
